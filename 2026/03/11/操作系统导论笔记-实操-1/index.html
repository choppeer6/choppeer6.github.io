操作系统导论笔记+实操——1

引言

《操作系统导论》（Operating Systems: Three Easy Pieces, OSTEP）是操作系统领域的经典教材，它不仅理论讲解透彻，还配备了丰富的代码示例和实验。这本书提倡“学以致用”，因此配套的 p1.c 等用户态程序能帮助我们直观感受系统调用的行为。更进一步，MIT 的 xv6 教学系统则提供了一个迷你内核，让我们能深入操作系统内部，亲眼见证进程、内存、文件系统等抽象的实现。

本文档将记录我从零开始学习 OSTEP 的实践过程，包括如何运行第一个示例程序，以及认识 xv6 这个宝贵的“实验田”。

第一部分：运行第一个程序 —— p1.c

1.1 环境准备

OSTEP 中的示例代码使用了 Linux 特有的系统调用（如 fork()），因此必须在 Linux 或类 UNIX 环境下运行。

  • Windows 用户：最简单的方式是安装 WSL（Windows Subsystem for Linux），例如 Ubuntu 20.04/22.04。
  • macOS 用户：直接使用内置的终端即可，macOS 基于 UNIX，可以完美兼容。
  • Linux 用户：天生满足条件，直接打开终端。

第二部分：认识 xv6 —— 理论与实践的结合

2.1 什么是 xv6？

xv6 是麻省理工学院（MIT）开发的一个教学用操作系统内核，它基于早期 Unix 第六版（v6），用现代 C 语言重写，代码量仅一万行左右。它麻雀虽小五脏俱全，包含了进程、内存管理、文件系统、并发控制等核心模块，并且有一本配套的《xv6 手册》逐行解释代码。

  • 与 OSTEP 的关系：OSTEP 是“理论”，讲解操作系统的基本原理；xv6 是“实践”，让我们能亲手修改和运行一个真正的内核，验证书中的概念。

2.2 我需要下载 xv6 吗？

建议先不着急，学到关键章节时再动手。 当你在 OSTEP 中学习了进程调度、锁、文件系统实现等内容后，就可以打开 xv6 的源码，看看书中那些抽象是如何用代码实现的。

如果你想现在就开始体验，可以按照下面的步骤搭建 xv6 实验环境。

2.4 xv6 能做什么？

  • 作为学习平台：你可以修改内核代码，比如增加一个新的系统调用、改变进程调度算法、实现信号量等。每次修改后重新 make qemu，观察效果。这是理解操作系统最有效的方式。

  • 作为真实系统：xv6 支持基本的 Unix 命令，你也可以编写自己的 C 程序，通过系统调用（如 fork, exec, open）与内核交互。

  • 作为探索游乐场：许多学生将 xv6 移植到真实硬件，甚至有人基于它开发商业产品（如 T6 安全内核）。它的简洁性让它成为系统编程和操作系统研究的绝佳起点。

结语

从运行一个简单的 p1.c，到进入 xv6 的内核世界，我们迈出了操作系统实践的第一步。后续我会继续记录学习过程中的笔记和实验心得，希望能与同样在学习 OSTEP 的你交流进步。

Happy hacking!
