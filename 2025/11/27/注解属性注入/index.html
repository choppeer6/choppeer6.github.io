注解属性注入

发表于2025-11-27|更新于2025-11-27
|浏览量:

在看面经是看到这样一个问题：

​ 怎么在spring配置文件中配置map集合，然后在代码中获取，会用到什么注解？

本篇解决基于SpringBoot 不讨论SpringXML

当不是Map属性，而是基本数据结构怎样配置?

1. @Value注解

  • 可以绑定单个属性的值
  • 使用 ${key} 或 SpEL#{...}表达式
  • 适合简单的配置

2. 使用 @ConfigurationProperties：

  • 绑定一组相关的属性值到一个POJO类
  • 使用统一的前缀(prefix="...")
  • 通常需要标准的Get和Set方法 /@Data
  • 支持数据检化验 @NotNULL @Min

如何配置Map

1.使用ConfigurationProperties(推荐)

2.使用 @Value 和 SpEL（适用于单行定义或 Properties 文件）

SpEL

#{$...} 是 Spring 框架中结合使用两种核心语法的特殊写法：

  1. ${}：属性占位符（Property Placeholder）
  2. #{}：Spring 表达式语言（SpEL, Spring Expression Language）

这种写法的核心用途是：先从外部配置文件读取一个字符串值，然后将该字符串内容当作 SpEL 表达式来执行。

详细解释使用方法：@Value("#{$...}")

当您在 @Value 注解中使用 #{$...} 语法时，Spring 处理的流程分为两步：

第一步：解析内部 ${property.key}

Spring 容器会查找 application.properties 或 application.yml 等配置文件中对应的 property.key，提取其原始的字符串值。

第二步：执行外部 #{...}

获取到这个字符串值后，整个字符串会被视为一个 SpEL 表达式。SpEL 引擎会解析并执行这个表达式，将字符串转换为代码中需要的实际 Java 对象或执行特定的逻辑。

实际应用场景：注入 Map 集合

最常见且实用的 #{$...} 用法就是将配置文件中表示 Map 或 List 的字串转换为实际的 java.util.Map 或 java.util.List 对象。
