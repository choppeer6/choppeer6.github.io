Hexo文件编写

写 Hexo 文章超简单，核心流程是「用命令新建文章文件 → 在指定目录编辑 → 本地预览 → 部署生效」，一步步跟着来就行：

2. 找到文件并打开

直接去 source/_posts 目录找到刚生成的 .md 文件，用编辑器打开即可开始写：

  • 推荐编辑器：Typora（实时预览 Markdown 格式）、VS Code（需装 Markdown 插件）、记事本（极简可用）。

二、怎么写：文章结构与编辑技巧

每篇 Hexo 文章都是 Markdown 格式，核心分「头部配置」和「正文内容」两部分：

1. 先配置文章头部（Front-matter）

文件最上方的 --- 之间是配置区域，控制文章的标题、标签、分类等，必须保留且格式正确（冒号后加空格）：

2. 写正文：常用 Markdown 语法（直接复制即用）

Markdown 语法简单，记住这些常用的就够了，配合快捷键更高效：

效果需求 Markdown 语法 快捷键（Typora）
一级标题 # 一级标题（# 后加空格） Ctrl+1
粗体 **这是粗体** Ctrl+B
斜体 *这是斜体* Ctrl+I
引用 > 这段是引用内容 Ctrl+Shift+Q
代码块（多行） bash（换行写代码） Ctrl+Shift+K
有序列表 1. 列表项1 2. 列表项2 Ctrl+Shift+(
无序列表 - 列表项 或 * 列表项 Ctrl+Shift+)
超链接 [链接文字](https://xxx.com "描述") Ctrl+K
插入图片 ![图片描述](图片文件名)（图片放同名文件夹） Ctrl+Shift+I
分割线 ---（单独一行） 直接输入后回车

3. 实用技巧：让文章更规范

  • 首页只显示摘要：在正文想截断的地方插入 <!--more-->，首页只会显示该标志前的内容，点击文章才看完整内容。
  • 插入本地图片：直接把图片放进 source/_posts 下的「文章同名文件夹」，然后用 ![描述](图片名.jpg) 引用即可。
  • 修改文章模板：如果想让每次新建文章都自带固定配置（比如默认标签），可以编辑 scaffolds/post.md 文件，修改里面的 Front-matter 模板。

三、写完后：预览与部署

1. 本地预览（确认效果再发布）

打开浏览器访问 http://localhost:4000，就能看到文章的最终效果，有问题可以直接修改 .md 文件，刷新页面就会同步更新。
